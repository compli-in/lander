## Part 4: Type I vs Type II

SOC 2 has two report types.

Most teams treat them as levels.

They are not.

They test fundamentally different things.

### Type I: Point-in-Time Validation

Type I answers one question:

Do your controls exist right now?

Auditors check:

* Whether controls are defined  
* Whether they are implemented  
* Whether they align with chosen criteria  

This is a snapshot.

If everything looks correct at that moment, you pass.

### Type II: Continuous Validation

Type II answers a harder question:

Do your controls actually operate over time?

Auditors check:

* Whether controls are executed consistently  
* Whether deviations occur  
* Whether issues are detected and resolved  

This is not a snapshot.

This is behavior over time.

### The Gap Between Type I and Type II

Most companies underestimate this gap.

Type I can be achieved with:

* Documentation  
* One-time fixes  
* Temporary alignment  

Type II requires:

* Ongoing enforcement  
* Continuous monitoring  
* System-level consistency  

This is where most teams break.

### Where Companies Fail

Common patterns:

* Controls implemented only for audit  
* Evidence collected manually  
* No system to ensure consistency  
* Gaps between teams and systems  

What passes in Type I does not survive Type II.

### The Evidence Problem

Type II is evidence-heavy.

Auditors expect:

* Logs over time  
* Records of control execution  
* Proof of monitoring and remediation  

If evidence is missing for any period, controls fail for that period.

### Operational Reality

Type II is not about passing an audit.

It is about sustaining behavior.

This means:

* Controls must run without manual intervention  
* Failures must be detected automatically  
* Corrections must be logged and provable  

Anything manual will break over time.

### The Only Useful Model

Type I proves you can set up controls.

Type II proves your system can sustain them.

If your controls depend on people remembering,  
Type II will expose it.

SOC 2 does not reward preparation.

It rewards consistency.
